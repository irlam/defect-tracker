1. Introduction

The Website Backup System provides a simple way to create and manage backups of your entire website, including files and database. This guide will walk you through all the features of the backup system.

Key Features:

  • One-click manual backups with progress tracking
  • Scheduled automatic backups (daily, weekly, or monthly)
  • Full website and database backups
  • Ability to restore the entire site or individual files
  • Download backups for offline storage
  • Automatic management of backup storage (keeps the most recent 10 backups)

2. Creating a Manual Backup

Steps to Create a Backup:

  1. Log in to the Backup System at https://yourcompanyname.defecttracker.uk/backups/
  2. On the main dashboard, click the blue "Create Full Backup" button
  3. Confirm the backup when prompted
  4. Wait for the backup to complete - you'll see a progress bar showing:
    • The current percentage complete
    • What part of the backup is currently running
    • Which files are being processed
  5. Once complete, the backup will appear in the "Available Backups" list

Note: The backup process may take several minutes depending on the size of your website. Do not close the browser window while the backup is in progress.

What Happens During a Backup:

  1. The system first backs up your database (all tables and data)
  2. It then creates a ZIP archive and starts adding website files
  3. The database backup is added to the ZIP file in a "database" folder
  4. The completed backup is stored in the backups directory
  5. The system maintains a maximum of 10 backups. When this limit is exceeded, the oldest backup is automatically deleted.

3. Setting Up Scheduled Backups

Creating a Backup Schedule:

  1. Click the "New Schedule" button in the Scheduled Backups section
  2. Enter a name for your schedule (e.g., "Nightly Backup")
  3. Select a frequency:
    • Daily: Runs every day at the specified time
    • Weekly: Runs once a week on the selected day
    • Monthly: Runs once a month on the selected day
  4. Select the time of day for the backup to run (in 24-hour format)
  5. Make sure the "Enabled" checkbox is checked
  6. Click "Save Schedule"

Setting Up the Cron Job:

For scheduled backups to work, a cron job must be set up on the server:

  1. Click the "Set Up Scheduled Backups" button
  2. Follow the instructions specific to our Netcup hosting environment
  3. Add the provided cron command to run every 10 minutes
  4. Once set up, the scheduler will automatically run backups at your specified times

Note: It's recommended to schedule backups during low-traffic periods (like overnight) to minimize any impact on site performance. Remember that the system keeps a maximum of 10 backups, so plan your scheduling accordingly.

4. What's Included in a Backup

Each backup is a complete snapshot of your website and includes:

Files:

  • All website files (HTML, PHP, CSS, JavaScript, etc.)
  • Media files (images, videos, documents)
  • Configuration files
  • Theme files

Database:

  • Complete SQL dump of your database
  • All tables and data
  • Table structures and relationships

Excluded Items:

To keep backups efficient, these items are excluded:

  • Temporary files
  • Cache files
  • Log files
  • The backup system itself (to avoid recursion)

Backup Format:

Backups are stored as ZIP files with this format:

mcgoff-backup-YYYY-MM-DD_HH-MM-SS.zip

Inside each ZIP file, you'll find:

  • All website files in their original directory structure
  • A /database/ folder containing the SQL dump file

5. Restoring from a Backup

Warning: Restoring a backup will overwrite existing data. Make sure you understand what you're restoring before proceeding.

To Restore the Entire Website:

  1. Go to the "Available Backups" list
  2. Click "Browse Files" next to the backup you want to restore
  3. On the restore page, click the "Restore Full Backup" button
  4. Confirm the restoration when prompted
  5. Wait for the restore process to complete

To Restore Only the Database:

  1. Go to the "Available Backups" list
  2. Click "Browse Files" next to the backup you want to restore
  3. Click the "Restore Database" button
  4. Confirm the restoration when prompted

To Restore Individual Files:

  1. Go to the "Available Backups" list
  2. Click "Browse Files" next to the backup you want to restore
  3. Locate the file you want to restore in the list
  4. Click "Restore This File" next to the file

Note: You can use the search function at the top of the file list to quickly find specific files.

6. Browsing and Downloading Backups

Viewing Available Backups:

All your backups are listed in the "Available Backups" section of the main page, showing:

  • Filename (including date and time)
  • Creation date
  • File size
  • Available actions

Backup Retention:

The system is configured to keep a maximum of 10 backups. When the 11th backup is created, the oldest backup is automatically removed to save storage space. If you need to keep a backup permanently, download it to your local system for archiving.

Downloading a Backup:

  1. In the "Available Backups" list, click the "Download" button next to the backup you want
  2. The backup will be downloaded to your computer as a ZIP file
  3. Store this file in a safe location for off-site backup

Browsing Backup Contents:

  1. Click the "Browse Files" button next to any backup
  2. You'll see a complete list of all files included in the backup
  3. Use the search box to filter files by name or path

7. Troubleshooting

Common Issues and Solutions:

Issue Possible Solution
Backup fails with timeout error The website may be too large for a single backup process. Try excluding larger directories in the configuration.
Scheduled backups not running Check that the cron job is set up correctly. View the scheduler log file for detailed information.
Database restore fails Ensure the database credentials are correct in the config.php file.
"Permission denied" errors Check that the web server has sufficient permissions to read and write to all necessary directories.
Backup is incomplete Run the System Diagnostics to check for any issues with your server configuration.
Need to keep more than 10 backups Download important backups to your local computer. The system automatically removes the oldest backups when the limit of 10 is reached.

Running Diagnostics:

The system includes a diagnostics tool to help identify issues:

  1. Click the "Run System Diagnostics" button on the main page
  2. The diagnostics page will show:
    • PHP version and configuration
    • Required PHP extensions
    • Database connection status
    • Availability of mysqldump command
    • Directory permissions
